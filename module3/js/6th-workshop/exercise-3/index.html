Closures en Acción

¿Cómo mantienen las funciones su acceso a variables externas después de que la función externa ha terminado de ejecutarse?

  • Mantienen el acceso ya que las funciones encadenan consigo la variables accedidas de los scopes superiores donde fueron definidas.

¿Cuáles son las implicaciones de memoria de mantener estos closures, especialmente si se crean muchas instancias de funciones con closures?

  • Por cada instancia de un contador creado se está creando una nueva función que ejecuta la lógica de aumentar el contador. Para lo cual, creando diferentes instancias de contador no reutilizará la primera funcion de incremento creada por el primer contador si no que crearan su propias funciones de incremento aumentando así el uso de memoría.
